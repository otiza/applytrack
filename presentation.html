2. Problématique

Le suivi de recherche d’emploi est souvent chaotique

  • Les utilisateurs postulent à plusieurs entreprises en même temps.
  • Le statut des candidatures se perd facilement : postulé, entretien, offre.
  • Les contacts recruteurs sont dispersés dans les messages et les notes.
  • Les dates d’entretien sont oubliées ou difficiles à organiser.
  • La version du CV utilisée pour chaque candidature n’est pas toujours claire.
  • Les relances et les notes de feedback ne sont pas centralisées.
3. Solution

ApplyTrack : un système de type CRM pour les chercheurs d’emploi

ApplyTrack centralise les entreprises, les candidatures, les entretiens, les CV, les notes et la progression du pipeline dans une plateforme claire, avec des données sécurisées et propres à chaque utilisateur.

Organiser

Suivre toutes les opportunités, de la liste de souhaits à la décision finale.

Prioriser

Se concentrer sur les candidatures importantes et les entretiens à venir.

Décider

Utiliser les statistiques et l’historique pour améliorer sa stratégie de recherche d’emploi.

9. Pipeline Kanban

Workflow visuel des statuts avec glisser-déposer

Liste de souhaits → Postulé → Entretien → Test technique → Offre / Refusé

Construit avec @dnd-kit. Les actions de glisser-déposer mettent immédiatement à jour le statut dans la base de données pour assurer la persistance entre les sessions.

10. Fonctionnalité de gestion des CV

Bibliothèque de CV + association aux candidatures

  • L’utilisateur peut téléverser plusieurs fichiers CV.
  • Validation : PDF/DOC/DOCX, avec limite de taille appliquée.
  • Les fichiers sont stockés dans le dossier uploads du backend.
  • Chaque candidature peut référencer un seul CV.
  • Utile pour savoir exactement quel CV a été envoyé.
  • Téléchargement et suppression depuis la bibliothèque personnelle de CV.
11. Calendrier des entretiens

Planifier les entretiens avec une visibilité par date

  • Les candidatures avec interviewDate apparaissent dans la vue calendrier.
  • Met en évidence les entretiens à venir pour mieux prioriser.
  • Cliquer sur un événement ouvre les détails de la candidature liée.
  • Inclut des états de chargement et des états vides pour une UX propre.
12. Décisions techniques

Pourquoi ces choix ?

React + TypeScript : interface évolutive avec sécurité de typage.

Express : couche API simple et flexible.

Prisma + PostgreSQL : modélisation relationnelle et requêtes sécurisées.

JWT : authentification stateless et pratique pour une SPA.

Docker : environnements cohérents pour le développement et la démonstration.

Tailwind CSS : système d’interface moderne et rapide.

@dnd-kit : interactions de glisser-déposer accessibles.

13. Sécurité et validation

Protection des données et intégrité

  • Hachage des mots de passe avec bcrypt.
  • Routes protégées par JWT et middleware d’authentification.
  • Vérification de la propriété utilisateur sur chaque ressource.
  • Validation des payloads API avec les schémas Zod.
  • Validation des fichiers téléversés : type et taille.
  • Les utilisateurs ne peuvent accéder qu’à leurs propres données.
14. Défis

Principales difficultés d’implémentation

Gérer de manière fiable l’état d’authentification côté frontend.

Persister correctement les changements du pipeline par glisser-déposer.

Lier les candidatures aux entreprises et aux documents CV.

Gérer les téléversements de fichiers et le contrôle d’accès sécurisé.

Maintenir une configuration Docker stable pour tous les services.

Équilibrer une UX claire avec des workflows riches en fonctionnalités.

15. Scénario de démonstration

Script de démo en direct

  1. S’inscrire ou se connecter.
  2. Créer une entreprise.
  3. Téléverser un CV dans la bibliothèque de CV.
  4. Créer une nouvelle candidature.
  5. Associer le CV téléversé à cette candidature.
  6. Déplacer la candidature dans le tableau Kanban.
  7. Ajouter une date d’entretien.
  8. Ouvrir le calendrier et vérifier l’événement d’entretien.
  9. Consulter les statistiques du tableau de bord.
  10. Ajouter des notes à la candidature.
